THE ABYSS

A Procedurally Generated Dungeon Crawler

Python Final Project  ·  2025–26

What is The Abyss?

A top-down dungeon crawler where every floor is randomly generated — no two playthroughs are ever the same.

🗺️

Procedural Maps

Rooms and corridors are carved randomly each time you play

👹

3 Enemy Types

Goblins, Skeletons and Demons — each with unique stats

🕯️

Torch Lighting

You can only see what your torch reveals — darkness hides danger

📈

Gets Harder

Enemies gain HP and spawn in larger groups on deeper floors

How It Works

🏗️

Step 1 — Build the Map

Start with solid rock. Punch in random rooms. Dig corridors to connect them. Place enemies and loot.

🧠

Step 2 — Enemy AI

Enemies patrol until they spot you. Then they calculate the shortest path around walls and give chase.

🎨

Step 3 — Render

Camera follows the player. Torch light is drawn over the whole screen every frame.

All of this runs 60 times per second inside a single game loop

Procedural Generation

The map is different every single time. Here's how it's built:

1 · Place Rooms

Try 400 random positions. If a room doesn't overlap anything already there, keep it. Repeat up to 16 rooms.

2 · Dig Corridors

Sort the rooms left to right. Carve an L-shaped tunnel between each pair — so every room is reachable.

3 · Populate

Player starts in room 1. Stairs down are in the last room. Enemies and items fill all rooms in between.

Uses a 2D list of integers — 0 = wall, 1 = floor, 2 = stairs — the core data structure of the entire game

Enemy AI

Enemies aren't mindless — they can only chase you if they can actually see you.

👁️

Line of Sight

An invisible line is drawn from the enemy to the player. If a wall crosses it — the enemy can't see you and stays idle.

🗺️

Pathfinding (BFS)

Once spotted, the enemy runs a Breadth-First Search — the same algorithm used in GPS and robot navigation — to find the shortest route around walls.

⚡

Performance

Recalculating paths every frame would be too slow. Instead I cache the path and only recalculate every 0.48 seconds.

Data Structures

📋 Lists of Lists

The whole map is stored as a 54×42 grid — a list where each row is another list of numbers. Fast to look up any tile by its x/y position.

📦 Dictionaries

Every enemy is a dictionary: hp, speed, position, path, state. All 20+ enemies live in one list of dicts — easy to loop, update, and render.

👤 Player Dict

One dictionary holds everything about the player: position, health, attack timer, facing direction, alive/dead.

✨ Particle List

Kill effects and pickup sparks each live as a dict in a list. Removed automatically when their lifetime runs out.

Challenges & How I Fixed Them

Problem Black square around the player — the torch light only darkened a small rectangle instead of the whole screen

Fixed Fill the entire screen black, draw the light circle on it, then multiply over the whole frame

Problem Wall misalignment — the player could visually walk into walls because physics and visuals used different positions

Fixed Spawn everything at tile + 0.5 so the physics position is always the visual centre

Problem Performance — running pathfinding for every enemy every frame caused the game to stutter

Fixed Cache the path and only recalculate it every 0.48 s — 97% cheaper, still feels instant

What Is The Abyss?

A top-down dungeon crawler where every floor is uniquely generated — no two playthroughs are the same. Navigate torch-lit halls, fight monsters, collect loot, and descend as deep as you dare.

🗺️

Procedural Maps

Random rooms carved out of solid rock, connected by hand-dug corridors

👹

3 Enemy Types

Goblins, Skeletons & Demons — each with unique stats and hunting behaviour

🕯️

Torch Lighting

Radial light mask limits your field of view — darkness hides what lurks ahead

📜

Infinite Floors

Enemies scale with depth — floor 1 is manageable, floor 10 is brutal

How It Works

Three core systems

🏗️

1 · Generation

Random rooms placed on a grid → L-shaped corridors carved between them → enemies and items scattered through rooms

🧠

2 · Enemy AI

BFS shortest-path finds routes through walls → ray-marching checks line of sight → enemies hunt only when they can see you

🎨

3 · Rendering

Camera tracks player → tiles drawn with depth-based darkening → torch mask multiplied over the entire frame each tick

All three systems run inside a single 60 FPS game loop — events → logic → render → display

Procedural Generation

The map starts as solid wall tiles. Rooms are punched in at random, then connected by corridors — guaranteeing every room is reachable. 


def generate_dungeon(floor_num):
    """Procedurally build a floor — rooms, corridors,
    enemies, and items all in one pass."""

    tiles = [[WALL] * MAP_W for _ in range(MAP_H)]
    rooms = []

    # Place up to 16 non-overlapping rooms
    for _ in range(400):
        rw = random.randint(4, 12)
        rh = random.randint(4, 8)
        rx = random.randint(1, MAP_W - rw - 1)
        ry = random.randint(1, MAP_H - rh - 1)

        # Skip if it overlaps an existing room
        if any(rx < r['x']+r['w']+1 and ...
               for r in rooms):
            continue

        rooms.append({'x':rx,'y':ry,'w':rw,'h':rh})
        for ty in range(ry, ry + rh):
            for tx in range(rx, rx + rw):
                tiles[ty][tx] = FLOOR   # carve room

Step 1 — Rooms

Try 400 random placements. Accept if no overlap. Stop at 16 rooms.

Step 2 — Corridors

Sort rooms left→right. Carve 2-tile-wide L-shaped paths between each adjacent pair.

Step 3 — Populate

Place player in room[0], stairs in last room, enemies and loot in all rooms between.

Enemy AI

Enemies aren't telepathic — they can only chase you if they have line of sight. Once they see you, BFS finds the shortest walkable path. 


def bfs_path(tiles, start, goal, max_dist=28):
    """Breadth-first search through non-wall tiles.
    Returns the shortest path as a list of coords."""

    q       = deque([(sx, sy, [])])
    visited = {(sx, sy)}

    while q:
        x, y, path = q.popleft()
        for dx, dy in ((1,0),(-1,0),(0,1),(0,-1)):
            nx, ny = x+dx, y+dy
            if tiles[ny][nx] == WALL: continue
            if nx == gx and ny == gy:
                return path + [(nx, ny)]  # found!
            q.append((nx, ny, path+[(nx,ny)]))
    return []   # unreachable
    

def has_los(tiles, ax, ay, bx, by):
    """Ray-march from A to B. Return False
    if any wall tile intersects the line."""

    dist  = math.hypot(bx-ax, by-ay)
    steps = max(2, int(dist * 3))

    for i in range(steps + 1):
        t  = i / steps
        tx = int(ax + (bx - ax) * t)
        ty = int(ay + (by - ay) * t)
        if tiles[ty][tx] == WALL:
            return False   # blocked
    return True            # clear

BFS is recalculated every 0.48 s per enemy — cheaper than every frame, still feels reactive

Data Structures

Lists and Dicts doing the heavy lifting

🗂️ Tile Grid

list[list[int]]

54 × 42 grid — each cell is WALL=0, FLOOR=1, or STAIRS=2. O(1) lookup by tile coordinate.

👤 Player State

dict

x, y, hp, max_hp, facing, atk_t, atk_cd, inv_t, alive — everything about the player in one place

👹 Enemy List

list[dict]

Each enemy is a dict with its own type, hp, path, state, facing. Looped each frame for AI, rendering, and combat.

✨ Particles

list[dict]

Kill and pickup effects — each particle stores x, y, vx, vy, life, col, size. Removed when life ≤ 0.

Challenges & Solutions

Challenge Black square around the player — the torch light mask was a fixed-size surface blitted directly onto the game, leaving visible dark corners outside the circle

Fix Fill a full-screen black surface, blit the white radial gradient onto it at the player's position, then multiply it over the entire frame using BLEND_RGB_MULT

Challenge Wall collision misalignment — player's physics position was at the tile's top-left corner, but the sprite was rendered half a tile ahead, causing visual overlap into walls

Fix Spawn all entities at tile + 0.5 so their physics position is their visual centre — removed all the manual offset adjustments

Challenge Enemy pathfinding performance — running BFS for every enemy every single frame would freeze the game on busy floors

Fix Cache each enemy's path and only recompute it every 0.48 seconds — still feels responsive but costs ~8% of the CPU time
